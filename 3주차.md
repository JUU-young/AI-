

## 깃(git)
---
git(깃)은 분산 버전 관리 시스템(DVCS)으로, 소프트웨어 개발 과정에서 소스 코드의 변경 사항을 추적하고 관리하는 데 사용된다. git을 사용하면 여러 개발자가 동시에 작업할 수 있으며, 변경 내용을 효과적으로 병합하고 관리할 수 있다.

#### Git의 주요 특징
- **분산 버전 관리 시스템**: GIt은 중앙 서버 없이도 각 개발자들이 자신의 로컬 저장소에서 독립적으로 작업할 수 있다. 모든 히스트로리를 로컬에 보관하기 때문에 네트워크 연결 없이도 대부분의 작업을 수행할 수 있다.
- **효율적인 브랜칭과 병합**: Git은 브랜치를 쉽게 생성하고 병합할 수 있도록 설계되었다. 브랜치를 통해 새로운 기능을 개발하거나 버그를 수정하는 등의 작업을 독립적으로 진행할 수 있으며, 이후에 메인 프로젝트에 병합할 수 있다.
- **빠른 속도**: Git 저장소는 효율적으로 데이터를 저장하여, 디스크 공간을 적게 차지한다. 이는 Git이 변경된 파일의 전체 복사본을 저장하는 대신, 변경 사항만을 기록하기 때문이다.
- **데이터 무결성**: Git은 데이터를 저장할 때마다 체크섬을 사용하여 무결성을 보장한다. 각 파일의 변경 사항은 SHA-1 해시를 통해 고유하게 식별되며, 이로 인해 데이터의 손상이나 변조를 쉽게 감지할 수 있다.

#### Git의 기본 용어
- **저장소(Repository)**: 프로젝트의 모든 파일과 변경 이력을 저장하는 장소다. 로컬 저장소와 원격 저장소가 있다.
- **커밋(Commit)**: 벼녕 사항을 기록하는 행위로, 특정 시점의 프로젝트 상태를 저장한다.
- **브랜치(Branch)**: 독립적인 작업 공간으로, 새로운 기능 개발이나 버그 수정 등을 위해 사용된다.
- **머지(Merge)**: 브랜치를 합치는 작업으로, 여러 브랜치에서의 변경 사항을 하나로 통합한다.
- **풀(Pull)**: 원격 저강소의 변경 사항을 로컬 저장소로 가져오는 작업이다.
- 푸시(Push): 로컬 저장소의 변경 사항을 원격 저장소에 업로드하는 작업이다.
## 깃허브(GitHub)
![[Pasted image 20260517221727.png]]

- GitHub는 Git 저장소를 호스팅하고 협업할 수 있는 웹 기반 플랫폼이다. 2008년 출시된 이후 전세계에서 가장 많이 사용되는 소스 코드 관리 플랫폼으로 성장했으며, 2018년 Microsoft에 인수되었다.

#### GitHub란 무엇인가?

깃허브는 크게 두가지 측면에서 활용된다.

**Git 저장소 호스팅 서비스**

GitHub의 기본 기능은 Git 저장소를 클라우드에 저장하고 관리하는 것이다. 로컬에서 GIt으로 관리하던 코드를 GitHub에 업로드하면, 어디서든 접근할 수 있고 다른 사람과 공유할 수 있다. 무로 계정으로 무제한의 공개 저장소와 비공개 저장소를 만들 수 있다.

**프로그램 소스 코드 버전을 관리하는 이유**

프로그램의 소스 코드는 프로그램의 핵심 요소이자 프로그램을 소유한 기업의 자산이라고 할 수 있다. 때문에 기업들은 프로젝트 진행시 프로그램의 코드를 백업해서 분실 위험을 최소화하고, 코드 수정시 오류가 발생할 상황을 대비하고 있다.

**GItHub를 많이 쓰는 이유**

실무에서 GitHub(또는 GitLab, bitbucket 등의 중앙 저장소 서비스)를 이용하는 경우가 많은데, 그 이유는 이렇다.

1. 프로젝트 가시성
- 누가 어떤 작업을 하고 있는지 한눈에 파악할 수 있다
- 프로젝트의 전반적인 진행 상황을 추적할 수 있다
- 신규 참여자가 프로젝트를 쉽게 이해할 수 있다
- Pull Request를 통한 체계적인 코드 리뷰가 가능하다.

2. 품질 보증과 자동화
- 변경사항을 배포 전에 검증할 수 있다
- CI/CD를 통한 자동화된 테스트로 회귀를 방지할 수 있다.
- 코드 리뷰로 버그를 사전에 발견할 수 있다

3. 지식 공유와 문서화 코드와 문서가 체계적으로 축적된다.
- 이슈 트래커를 통해 결정 사항과 그 이유가 기록으로 남는다
- 팀의 노하우가 자연스럽게 공유된다

4. 백업과 장애 복구
- 중앙 저장소가 자동으로 백업 역할을 한다
- 서버 장애 시에도 쉽게 복구할 수 있다
- 여러 개발자가 전체 이력을 보유하고 있어 데이터 손실이 위험이 낮다\
