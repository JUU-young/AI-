## Linux
---
리눅스는 1991년 핀란드 소프트웨어 엔지니어 Linus Torvalds가 Unix를 기반으로 개발한 오픈소스 커널이다.
리눅스는 소스코드가 공개된 오픈소스 프로젝트로 누구나 수정 및 재배포가 가능하며 전세계 개발자들에 의해 지속적으로 발전됬따.

#### 운영체제
CPU, RAM 등의 HW 자원을 관리하고 응용 SW가 HW 자원을 사용할 수 있도록 중재하는 시스템 이다. 
메모리 할당, CPU 점유, 입출력 장치 등의 관리를 담당

####  커널
운영체제의 핵심 기능을 담당하는 구성 요소, 커널과 시스템 라이브러리, 명령어 도구 등이 합쳐져 운영체제를 구성한다.

#### Linux 기반 운영체제
리눅스 커널에 시스템 라이브러리, 패키지 관리자, 기본 유틸리티 등을 함께 구성한 형태를 린눅스 배포판이라고 한다. (예: Ununtu, Debian)

- Jetson Orin Nano는 Ununtu 기반 사용자 공간을 사용하며, 여기에 보드 전용 드라이버오 BSP가 포함되 Jetson Linux 환경에서 동작한다.
- 리눅스는 필요한 구성요소만 선택적으로 설치할 수 있어 시스템을 가볍고 효율적으로 구성할 수 있으며, 센서, 전원 제어 등 하드웨어를 다루기 위한 도구가 잘 갖춰져 있어 임베디드 시스템에 적합하다.
- 오픈소스 기반이므로 내부 구성을 목저에 맞게 수정하기 쉽고, 장시간 안정적으로 동작하는 시스템을 구현하는데 유리하다.

#### Windows vs LInux
Windows와 Linux는 모두 운영체제이지만, Windows는 GUI 중심의 사용자 친화성과 상용 소프트웨어 호환성에 강하고, Linux는 오픈소스 기반의 유연한 커스터마이징과 개발·서버·임베디드 환경에 강하다.
또한 Windows는 통합된 제품 형태로 배포되는 반면, Linux는 리눅스 커널을 기반으로 다양한 배포판이 존재하며 필요한 기능만 선택적으 로 구성할 수 있다.

|        | Windowx               | LInux                     |
| ------ | --------------------- | ------------------------- |
| 성격     | 성격 상용 통합 운영체제         | 오픈소스 커널 기반 운영체제 계열        |
| 배포 방식  | Microsoft가 통합 배포      | 다양한 배포판 존재                |
| 사용성    | GUI 중심, 일반 사용자 친화적    | CLI/GUI 모두 가능, 제어와 자동화 강함 |
| 소스 공개  | 비공개                   | 공개                        |
| 설치 방식  | exe, msi 중심           | 패키지 관리자 중심                |
| 주요 강점  | 호환성, GUI, 게임, 상용 프로그램 | 서버, 개발, 임베디드, 안정성, 유연성    |
| 주요 활용  | 개인 PC, 사무, 게임         | 서버, 클라우드, 연구, 임베디드        |
| 커스터마이징 | 제한적                   | 매우 높음                     |

#### Linux 디렉토리 구조
- Windows는 드라이브(C:\,D:\)별로 파일 시스템이 구분되는 반면, 리눅스는 최상단 루트 디렉토리 “ / ” 아래에 모든 파일과 디렉토리가 연결 되는 계층적 트리 구조를 가진다.
-  쉘 세션은 현재 작업 위치 정보를 유지하며, 사용자는 이를 기준으로 파일에 접근하고 경로를 이동하며 작업을 수행한다.
- 파일이나 폴더에 접근할 때 경로를 통해 접근할 수 있다. 최상위 루트 디렉토리로부터 시작하는 경로를 ‘전체 경로’ 라고 하며, 현재 작업 위치로부터 시작하는 경로를 ‘상대 경로’ 라고 한다
 ![[Pasted image 20260527194129.png]]

| 디렉토리  | 주요 역할 및 저장된 공간                             |
| ----- | ------------------------------------------ |
| /bin  | 기본적인 명령어 실행 파일들이 저장된 공간                    |
| /boot | 리눅스 시스템 부팅 및 운영체제 시작에 필수적인 핵심 정보가 위치한 폴더   |
| /etc  | 시스템 전반의 운영과 관련된 거의 모든 설정 파일이 보 관된 디렉토리     |
| /home | 각 사용자 계정별로 할당된 개인 작업 공간 및 데이터 저장 위치        |
| /lib  | 운영체제 및 각종 응용 프로그램 동작에 필요한 공유 라 이브러리 파일 저장소 |

| 디렉터리             | 설명                                     |
| ---------------- | -------------------------------------- |
| `/`              | 전체 파일 시스템의 시작점이 되는 최상위 루트 디렉터리         |
| `/bin`           | 기본적인 명령어 실행 파일이 저장되는 디렉터리              |
| `/sbin`          | 시스템 관리 및 관리자용 명령어가 저장되는 디렉터리           |
| `/etc`           | 시스템 설정 파일이 저장되는 디렉터리                   |
| `/home`          | 일반 사용자들의 홈 디렉터리가 위치하는 디렉터리             |
| `/root`          | root 사용자 관리자의 홈 디렉터리                   |
| `/boot`          | 부팅에 필요한 커널, 부트로더 관련 파일이 저장되는 디렉터리      |
| `/dev`           | 하드웨어 장치를 파일처럼 표현한 장치 파일이 위치하는 디렉터리     |
| `/proc`          | 프로세스와 커널 상태 정보를 제공하는 가상 디렉터리           |
| `/sys`           | 하드웨어와 커널 관련 정보를 제공하는 가상 디렉터리           |
| `/tmp`           | 임시 파일을 저장하는 디렉터리                       |
| `/var`           | 로그, 캐시, 스풀 등 자주 변경되는 데이터가 저장되는 디렉터리    |
| `/usr`           | 사용자용 프로그램, 라이브러리, 문서 등이 저장되는 디렉터리      |
| `/lib`, `/lib64` | 프로그램 실행에 필요한 공유 라이브러리가 저장되는 디렉터리       |
| `/media`         | USB, 외장하드 등 이동식 저장장치를 마운트할 때 사용하는 디렉터리 |
| `/mnt`           | 파일 시스템을 임시로 마운트할 때 사용하는 디렉터리           |
| `/opt`           | 추가 설치한 응용 소프트웨어 패키지가 저장되는 디렉터리         |
| `/run`           | 현재 실행 중인 시스템 상태 정보가 임시로 저장되는 디렉터리      |

## LLM
대규모 언어 모델(LLM)은 방대한 양의 데이터를 학습하여 자연어 및 기타 유형의 콘텐츠를 이해하고 생성하여 광범위한 작업을 수행할 수 있는 딥러닝 카테고리이다. LLM은 단어 시퀀스를 처리하고 텍스트의 패턴을 포착하는데 탁월한 신경망 아키텍처의 일종(트랜스포머)을 기반으로 구축된다.

LLM은 시퀀스의 다음 단어를 반복적으로 예측하는 거대한 통계 예측 기계로 작동하며, 텍스트의 패턴을 학습하고 그 패턴을 따르는 언어를 생성한다.

LLM의 개발은 2010년대 후반과 2020년대에 걸친 인공지능의 폭발적인 크게 기여했다. 자주 사용되는 LLM은 유명세를 얻어 생성형 AI를 대중적으로 알리게되는 계기가 되었다.

- 대표모델

| 모델       | 회사          | 설명                                                                                                   |
| -------- | ----------- | ---------------------------------------------------------------------------------------------------- |
| chat GPT | OpenAI      | LLM 대중화를 이끈 모델이다. 범용적인 추론 능력이 매우 뛰어나며, 코드 생성, 데이터 분석 등 다양한 분야에서 활용된다.                                |
| Claude   | Anthropic   | 수배만 토큰에 달하는 방대한 컨텍스트 윈도우(Context Window)를 지원하여 긴 논문이나 방대한 코드베이스를 한번에 분석하는 데 탁월하다.                    |
| Geminai  | Google      | 처음 공개 되었을때는 GPT에 비교되었으나 구글이 가진 기술력을 바탕으로 최근 많은 성능 개선을 이뤄냈다. 머티모달, 그중에서도 이미지 처리에서 매우 뛰어난 성능을 보여주고 있다. |
| DeepSeek | DeepSeek AI |                                                                                                      |

#### Ollama
Ollama는 로컬 환경에서 대규모 언어 모델(LLM)을 쉽게 실행할 수 있게 해주는 오픈소스 도구다.
ChatGPT 같은 AI를 개인 컴퓨터에서 직접 돌릴 수 있게 해주는 프로그램이다.
인프라 없이 개인 PC에서 곧바로 AI 챗 기능을 체험할수 있도록 설계되어 있다.

```
https://ollama.com/
```
- 공식 홈페이지

Ollama 자체는 chat GPT나 Geminai처럼 LLM 자체는 아니지만 Ollama를 설치하면 다양한 LLM을 사용할 수 있게된다.

#### Gemma
Gemma는 Google의 무료 개방형 소규모 언어 모델이다. 이는  Geminai와 동일한 기술로 제작되었으며 Geminai의 경량 버전이라 생각하면 된다.

Gemma 모델은 Gemini 모델보다 간결하기 때문에 노트북과 배포할 수 있다.


## 실습

#### 1. 경로 설정
```bash
mkir ollama
```

#### 2. ollama 설치
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

#### 3. llm 설치
```bash
ollama pull gemma3:1b
```

- 어떤 모델이 설치되었는지 확인법
```bash
ollama --version
ollama list
```

#### 4. llm 실행
```bash
ollama run gemma3:1b
```

#### 5. 응용
```
https://github.com/JUU-young/Jetson-orin-Nano-project
```
해당 링크에 깃허브에서 깃을 해준다. 이후 해당 코드들을 아래와 같은 방식으로 실행해 준다.

```bash
python3 파일 이름
```
- 해당 방식으로 터미널에서 실행하면 파이썬 코드가 실행된다.
- 파일에 있는 모든 파일들을 실행해 보아라.
